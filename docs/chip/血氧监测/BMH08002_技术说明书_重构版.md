## 第一章 产品概述

**核心要点**：集成反射式血氧传感器，直出血氧/心率/PI，支持 UART 与 IIC，关断低至 1µA。

### 1.1 产品简介

BMH08002 是**悠健电子**推出的**血氧测量模组**，主控采用 HOLTEK 资源丰富的 **HT66F4550 MCU**——其内部集成最高 2MHz 带宽的双运算放大器（双 OP）并配合 DAC 等外围电路，即可完成血氧测量功能。模组**集成反射式血氧传感器**，常应用于**手指末端血氧测量**。

一句话理解：把它焊到板上、供上 3.3V、用串口或 I²C 发几条命令，它就会直接告诉你**血氧饱和度（SpO₂）、心率（HR）、血流灌注指数（PI）**，无需你自己写任何算法。

### 1.2 核心优势

| 优势 | 说明（通俗） |
|---|---|
| ■ 低功耗设计 | 关断电流仅 **1µA**，非常适合电池供电设备 |
| ■ 集成反射式血氧传感器 | 传感器已封装在模组内，省去光学器件选型与调试 |
| ■ 支援 IIC / UART | 两种主流通信协议任选，极易集成到主控 |
| ■ 直接输出测量结果 | 直出血氧值、心率、灌注指数，**免算法开发、降成本** |

### 1.3 工作原理简述（非专家版）

1. 模组内部 LED 交替发出**红光**与**红外光**照射手指。
2. 血液对两种光的吸收比例随含氧量变化，光电二极管接收反射光。
3. 内部 OP + DAC + MCU 完成信号放大、采样与计算。
4. 通过 UART 或 IIC 把算好的 SpO₂ / HR / PI 发给你的主控。

## 第三章 应用场景说明

**核心要点**：主攻指尖血氧监测，适配可穿戴设备与健康异常预警类终端。

### 3.1 典型应用场景

| 场景 | 说明 |
|---|---|
| ■ 智能可穿戴设备 | 手表 / 手环 / 指环等，低功耗特性（关断 1µA）契合电池供电 |
| ■ 健康异常检测 | 居家血氧/心率长期监测、睡眠呼吸异常预警等 |
| ■ 手指末端血氧测量 | 模组默认与核心用途，集成反射式传感器，免外部光学设计 |

### 3.2 选型与适配提示

- **需要实时画波形**（OLED/TFT 屏）：选用**持续上传（PPG 波形）模式**，并在屏幕扫到末端时下发「更新波形控制参数」命令避免断痕。
- **仅需数值显示**（LED 屏血氧仪）：选用**定时发送模式**（默认，间隔 1000ms）即可。
- **主控轮询式**：选用**询问–响应模式**，由主控主动取数。

## 第五章 典型应用电路详解

**核心要点**：UART 直连无需上拉，IIC 总线经 10kΩ 上拉到 VDD；EN 内拉高默认使能。

### 5.1 接口连接说明（据原图文字还原）

原 PDF 电路图文字层为连接关系碎片，还原如下：

- **UART 接法**：模组 `TXD/SCL` → 目标板 `RXD`；模组 `RXD/SDA` → 目标板 `TXD`；交叉直连即可。
- **IIC 接法**：模组 `TXD/SCL` 作 **SCL** → 目标板 SCL；模组 `RXD/SDA` 作 **SDA** → 目标板 SDA。
- 原图标注：**「UART/IIC 介面（注：UART 通讯不需接上拉电阻）」**。

### 5.2 电源与使能

- **VIN（Pin1）** 接 3.3V 电源，**GND（Pin4）** 接电源地。
- **EN（Pin5）** 内部已上拉，**悬空即默认使能**；拉低（EN=0）进入关断，电流约 1µA。对功耗敏感时应主动拉低 EN 而非仅进休眠。

### 5.3 上拉电阻

- **UART 模式**：**无需上拉电阻**，直接交叉连接。
- **IIC 模式**：SCL、SDA 需各经 **10kΩ 上拉到 VDD**（原图碎片 `VDD 10K 10K VDD` 即指双线上拉）。

### 5.4 封装尺寸（两种封装）

单位：mm

| 参数 | BMH08002-1 / -2 | BMH08002-3 / -4 |
|---|---|---|
| 模块长 I | 27 | 27 |
| 模块宽 W | 15 | 11 |
| 接口间距 d | 2 | 2 |
| 传感器中心→上边缘 a | 12.5 | 6.5 |
| 传感器中心→左边缘 c | 13.5 | 13.5 |
| 传感器尺寸（长×宽）s | 11 × 5.1 | 13.5 × 5.1 |

### 5.5 快速配置步骤（示例）

> 以 **UART** 为例，帧格式见附录 A.1。模块上电后默认即「开始测量」。

```text
# 1) 上电（EN 悬空 = 默认使能，出厂默认开始测量）
# 2) 开始测量（可选，若已默认测量可跳过）
发送: 55 B1 01 00 00 00 B2 AA
# 3) 获取测量结果（询问-响应模式下需要；定时/波形模式自动上报）
发送: 55 B1 03 00 00 00 B4 AA
# 4) 结束测量
发送: 55 B1 01 00 00 01 B3 AA
# 5) 进入休眠（省电）；或拉低 EN 彻底关断（≈1µA）
发送: 55 B1 01 00 00 02 B4 AA
```

## 附录 A 通信协议与寄存器速查

### A.1 UART 命令帧格式与示例

**通用帧（8 字节）**：`Head | CMD | OP_Code | Addr | Data_H | Data_L | CheckSum | Tail`

```text
Byte0  Head     = 0x55
Byte1  CMD      = 0xB1
Byte2  OP_Code  = 操作码
Byte3  Addr     = 寄存器地址
Byte4  Data_H   = 数据高字节
Byte5  Data_L   = 数据低字节
Byte6  CheckSum = Byte1+Byte2+Byte3+Byte4+Byte5（低8位）
Byte7  Tail     = 0xAA
```

**常用命令示例**：

| 功能 | OP_Code | 示例帧 |
|---|---|---|
| 开始测量 | 01 / Data=00 | `55 B1 01 00 00 00 B2 AA` |
| 结束测量 | 01 / Data=01 | `55 B1 01 00 00 01 B3 AA` |
| 进入休眠 | 01 / Data=02 | `55 B1 01 00 00 02 B4 AA` |
| 更新波形参数 | 02 | `55 B1 02 00 00 00 B3 AA` |
| 获取测量结果 | 03 | `55 B1 03 00 00 00 B4 AA` |
| 写 EEPROM | 04 | `55 B1 04 01 00 03 B9 AA`（配模式） |
| 读 EEPROM | 05 | `55 B1 05 02 00 00 B8 AA`（查红外最小） |
| 校准手指灵敏度 | 06 | `55 B1 06 00 00 00 B7 AA` |

> 光强换算：`光强 = Data_H × 256 + Data_L`；手指侦测灵敏度仅与红外光最小/最大光强有关。
> 定时发送间隔：`1 = 4ms`，最小 12ms（小于 12ms 会被强制设为 12ms）。

### A.1.1 完整配置命令字节表（9.10–9.21，写/设置）

> 设置类 OP_Code 均为 `04`（写 EEPROM），查询类均为 `05`（读 EEPROM）；Addr 与第四章 4.4 寄存器映射一致。所有示例帧 CheckSum 已验算通过。

### A.2 IIC 读写时序

**写 EEPROM**（Command=0x03）：

```text
S → 0x64(W) → 0x03 → Addr → Data_H → Data_L → P
```

**读 EEPROM**（Command=0x04）：

```text
S → 0x64(W) → 0x04 → Addr → S → 0x65(R) → Data_H → Data_L → P
```

**读测量数据**（Command=0x00，返回 6 字节）：

```text
S → 0x64(W) → 0x00 → S → 0x65(R) → Status → SpO2 → HR → PI → HRV → PEC → P
```

**设置工作状态**（Command=0x01）：OP_Code `00`=开始测量，`01`=停止测量，`02`=进入休眠。

### A.3 测量数据返回帧格式（UART 主动上报，共 15 字节）

```text
Byte0   Head      = 0x55
Byte1   CMD       = 0xB0（数据上传）
Byte2   Status    = 00无手指 / 01有手指无数据 / 02有手指有数据
Byte3   SpO2      单位 %（直接读值，范围 35~99）
Byte4   HR        单位 BPM（直接读值，范围 30~250）
Byte5   PI        读值×0.1 = %（如 62 → 6.2%）
Byte6   HRV       单位 ms
Byte7   脉搏高度  范围 0~32
Byte8   心脏收缩时间 单位 ms
Byte9   ADC1_H    PPG波形1 高字节
Byte10  ADC1_L    PPG波形1 低字节
Byte11  ADC2_H    PPG波形2 高字节
Byte12  ADC2_L    PPG波形2 低字节
Byte13  CheckSum  = Byte1+...+Byte12（低8位）
Byte14  Tail      = 0xAA
```

> IIC 返回 6 字节顺序：Status → SpO2 → HR → PI → HRV → PEC，PEC 校验码 = (各字节之和) & 0xFF。
> **范围说明（源文档不一致，见第二章 2.2 编辑注）**：SpO₂ 通信字节允许 35–99%（性能指标 70–99%）；PI 读值 0–200（=0–20.0%，性能指标 0.5–25%）。

### A.4 ModeConfig（01h）位域定义

| Bit | 名称 | 含义 | 默认 |
|---|---|---|---|
| 7–5 | — | 未定义 | 0 |
| 4 | CTRL_LOGIC | 0=SW_LED1=IR亮、SW_LED2=RED亮；1=反之 | 0 |
