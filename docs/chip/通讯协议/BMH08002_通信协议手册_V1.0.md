## 0. 速读摘要

| 维度 | 结论 |
|---|---|
| **双总线** | 同一对引脚复用：**TXD/SCL**、**RXD/SDA**；可任选 UART 或 IIC 接入 |
| **UART** | 波特率 **38400**，8 数据位，**无校验**，1 停止位（8N1）；命令帧 **8 字节**：`55 B1 OP Addr DH DL CS AA` |
| **IIC** | 7 位从机地址 **0x64**（写 0xC8 / 读 0xC9）；命令字节 **0x00–0x05** |
| **校验** | UART 命令帧：`CS = (B1+B2+B3+B4+B5) & 0xFF`；UART 数据帧：`CS = (B1..B12) & 0xFF`；IIC：`PEC = (各数据字节之和) & 0xFF` |
| **首条命令** | 上电默认开始测量；获取/配置前先确认固件版本（见 1.2 版本风险） |
| **最关键坑** | ① IIC 命令码 **0x05 被两个功能复用**（靠读写方向区分）；② 原文 §10.6 的 PEC 公式把 `RED_DC_L` 误写成 `RED_DC_H`；③ ModeConfig 的 `M_DATA[1:0]` 原文说明有重复笔误 |

## 1. 协议总览与版本风险

### 1.1 模组与 MCU

- 模组型号：**BMH08002**（悠健电子），内部 MCU 为 **Holtek HT66F4550**（含最高 2MHz 带宽双 OP + DAC）。
- 支持 **UART / IIC** 双通讯协议，直接输出 SpO₂（血氧）、心率（HR）、血流灌注指数（PI），无需用户自行开发算法。
- 典型应用：智能可穿戴设备、健康异常检测。

### 1.2 ⚠️ 协议版本风险（务必先确认固件版本）

原文档 **Revision History** 明确记录：

| Ver | Date | Description |
|---|---|---|
| 1.0 | 2019/05/27 | Initial version |
| 1.1 | 2019/12/03 | Version 1.1 |
| 1.2 | 2020/02/18 | 修改 PI 的测量精度 |
| **1.3** | 2020/12/09 | **更改协议** |
| **1.4** | 2021/01/21 | **更改协议** |
| **1.5** | 2021/03/08 | **更改协议** |
| **1.6** | 2021/04/17 | **更改增加 IIC 协议** |

> **专家提示**：UART 协议在 1.3 / 1.4 / 1.5 连续三个版本被「更改协议」，且 1.6 才新增 IIC。本手册对应 **V1.6**。若你手中的模组固件不是 V1.6，命令码、帧格式、校验和范围可能不同，**务必先核对模块丝印版本与本文档一致**，否则会出现「命令识别错误 / 无回复」。

### 1.3 两套命令码空间相互独立

| 总线 | 帧头/地址 | 功能区分方式 |
|---|---|---|
| UART | 固定帧头 `0x55` + 命令字 `0xB1` | 由 **OP_Code（Byte2）** 区分 6 类功能 |
| IIC | 7 位从机地址 `0x64` | 由 **Command 字节** 区分（0x00–0x05） |

两套码值各自成体系、**互不冲突**（例如 UART 用 OP_Code=0x05 表示「读 EEPROM」，IIC 用 Command=0x04 表示「读 EEPROM」）。

## 2. 物理层：引脚、电气、应用电路

### 2.1 引脚定义（5 Pin）

| Pin | Name | Description | 协议角色 |
|---|---|---|---|
| 1 | **VIN** | 模组电源（典型 3.3V） | 电源 |
| 2 | **TXD/SCL** | UART 发送脚 / IIC 时钟线 SCL | UART-TX 或 IIC-SCL |
| 3 | **RXD/SDA** | UART 接收脚 / IIC 数据线 SDA | UART-RX 或 IIC-SDA |
| 4 | **GND** | 电源地 | 地 |
| 5 | **EN** | 使能脚，内部拉高，**默认使能** | 拉低进入关断（Shutdown 电流 1µA） |

> 注意：引脚 2/3 **硬件复用**，同一对线既可以是 UART 也可以是 IIC，由主机侧驱动方式决定（同一时刻只用一种总线）。

### 2.2 应用电路要点

```
        VDD
         │
       10K        (IIC 模式下 SCL/SDA 需上拉；UART 模式不需上拉)
         │
目标板   TXD/SDA ──┐
        RXD/SCL ──┤   BMH08002
        GND      ──┤
        VDD      ──┘
```

- 原文档明确注明：**「UART 通讯不需接上拉电阻」**。
- **IIC 模式**需在 SDA/SCL 上接上拉电阻（典型 10kΩ 至 VDD，见应用电路图）。
- UART 与 IIC 共用 TXD/RXD 引脚，二者不应同时使用。

### 2.3 电气特性（DC）

| Parameter | Test Conditions | Min | Typ | Max | Unit |
|---|---|---|---|---|---|
| Operating voltage | fSYS=fHIRC=8MHz | — | **3.3** | — | V |
| Storage Temperature | Vin=3.3V | -20 | — | 55 | ℃ |
| Operating Temperature | — | 5 | — | 40 | ℃ |
| Operating current | Vin=3.3V | — | 5 | 16 | mA |
| Standby current | Vin=3.3V | — | 114 | — | µA |
| Shut Down current | Vin=3.3V, **EN=0** | — | 1 | — | µA |

> **专家提示**：Operating voltage 仅给出 Typ=3.3V，**Min/Max 均为「—」**（原文档未给范围）。工程上建议严格使用 3.3V 供电，不要按普通 3.3V 器件去容忍 3.0–3.6V 宽压。休眠（Shut Down）通过将 **EN 拉低**实现，电流约 1µA；若对功耗敏感，应失能（拉低 EN）模块而非仅靠 UART/IIC「进入休眠」命令（后者休眠电流约 110µA）。

## 3. 通信可达存储映射

模组内部有一片 EEPROM，地址 **00h–0Bh** 对通信可读写，所有配置（工作模式、光强、发送间隔）落在此处。

### 3.1 EEPROM 地址表

| 地址 | 存储器名称 | 可读 | 可写 | 说明 |
|---|---|---|---|---|
| 00h | Reserve | √ | √ | 保留 |
| 01h | **ModeConfig** | √ | √ | 模式配置寄存器（见第 8 章） |
| 02h | IRMinLevel HighByte | √ | √ | 红外光最小光强（高字节） |
| 03h | IRMinLevel LowByte | √ | √ | 红外光最小光强（低字节） |
| 04h | IRMaxLevel HighByte | √ | √ | 红外光最大光强（高字节） |
| 05h | IRMaxLevel LowByte | √ | √ | 红外光最大光强（低字节） |
| 06h | REDMinLevel HighByte | √ | √ | 红光最小光强（高字节） |
| 07h | REDMinLevel LowByte | √ | √ | 红光最小光强（低字节） |
| 08h | REDMaxLevel HighByte | √ | √ | 红光最大光强（高字节） |
| 09h | REDMaxLevel LowByte | √ | √ | 红光最大光强（低字节） |
| 0Ah | SendInterval HighByte | √ | √ | 定时发送时间间隔（**仅 UART 场景生效**） |
| 0Bh | SendInterval LowByte | √ | √ | 定时发送时间间隔（**仅 UART 场景生效**） |
| FFh | — | — | — | —（表格末行占位） |

### 3.2 数值寄存器编码规则

- **光强类寄存器（02h–09h）**为 16 位无符号值，bit 图在原文档中全为「—」（即不按位定义，纯数值存储）。
- **光强值 = `Data_H × 256 + Data_L`**（16 位，范围 0–65535）。
- **SendInterval（0Ah/0Bh）**：16 位值，**单位换算：1 LSB = 4 ms**；出厂默认值 `FAh`（Byte 高 0x00、低 0xFA）→ `0x00FA = 250`，`250 × 4ms = 1000ms`。
- IR/RED 光强的出厂值「根据机构的不同会有不同的出厂值（Default）」——**不要假设固定默认值**，集成前应读取确认。

## 4. UART 通信协议

### 4.1 UART 物理配置

```
波特率：38400
数据位：8 bit
校验位：None
停止位：1 bit
```

即标准 **38400 / 8N1**。无硬件流控。

### 4.2 UART 命令帧格式

所有「写/查询」命令均为 **8 字节定长**帧：

| Byte | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|---|
| 字段 | Head | CMD | OP_Code | Addr | Data_H | Data_L | CheckSum1 | Tail |
| 典型值 | `0x55` | `0xB1` | 见 4.4 | 见各命令 | 见各命令 | 见各命令 | 见 4.3 | `0xAA` |

### 4.3 UART 校验和算法

```
CheckSum1 = (Byte1 + Byte2 + Byte3 + Byte4 + Byte5) & 0xFF
          = (CMD + OP_Code + Addr + Data_H + Data_L) & 0xFF
```

> 即帧内 **第 2～6 字节（索引 1–5）累加，取低 8 位**。Head(0x55) 与 Tail(0xAA) **不参与**校验和计算。
> ⚠️ 注意区分：命令帧的校验和是 `Byte1..Byte5`；而测量数据上传帧（§4.6）的校验和是 `Byte1..Byte12`，二者范围不同，勿混用。

### 4.4 UART OP_Code 命令总表

| OP_Code | 功能 | Addr 取值 | Data_H / Data_L | 是否返回数据 |
|---|---|---|---|---|
| `0x01` | 设置工作状态 | `0x00` | Data：`0`=开始测量，`1`=结束测量，`2`=进入休眠 | 否（回显） |
| `0x02` | 更新波形控制参数 | `0x00` | `0x00` / `0x00` | 否（回显） |
| `0x03` | 获取当前测量结果 | `0x00` | `0x00` / `0x00` | **是（15 字节数据帧）** |
| `0x04` | 写 EEPROM | 目标地址 | 数据高/低字节 | 否（回显） |
| `0x05` | 读 EEPROM | 目标地址 | `0x00` / `0x00` | **是（返回 EEPROM 数据）** |
| `0x06` | 校准侦测手指灵敏度 | `0x00` | `0x00` / `0x00` | 否（回显） |

> **回复规则（原文档原文）**：「每发一条命令，模块均会回复对应命令（仅获取数据命令会回复对应数据）。」
> 解读：模块对每条命令都会**回显原帧**；其中 `0x03`（获取结果）和 `0x05`（读 EEPROM）会在回显基础上额外返回对应数据。

### 4.5 UART 各命令详解与示例帧

> 下列示例帧的 `CS` 字节均已按 4.3 公式重算，与原文档完全一致（验证明细见第 9 章）。

#### 4.5.1 设置工作状态（OP_Code = 0x01）

| Data 值 | 含义 |
|---|---|
| `0x00` | 开始测量 |
| `0x01` | 结束测量 |
| `0x02` | 进入休眠 |

- 开始测量：`55 B1 01 00 00 00 B2 AA` → 回复 `55 B1 01 00 00 00 B2 AA`
- 结束测量：`55 B1 01 00 00 01 B3 AA` → 回复 `55 B1 01 00 00 01 B3 AA`
- 进入休眠：`55 B1 01 00 00 02 B4 AA` → 回复 `55 B1 01 00 00 02 B4 AA`

> 校验和验算：开始=`B1+01+00+00+00=0xB2` ✓；结束=`B1+01+00+00+01=0xB3` ✓；休眠=`B1+01+00+00+02=0xB4` ✓。

#### 4.5.2 更新波形控制参数（OP_Code = 0x02）

- 命令：`55 B1 02 00 00 00 B3 AA` → 回复 `55 B1 02 00 00 00 B3 AA`
- **用途**：用于需要显示一段时间内多个脉搏波波形的场合（OLED/TFT 屏）。在屏幕扫描到末端时下发此命令刷新波形控制参数，**避免中途更新造成波形断痕**。
- 校验和：`B1+02+00+00+00=0xB3` ✓。

#### 4.5.3 获取当前测量结果（OP_Code = 0x03）

- 命令：`55 B1 03 00 00 00 B4 AA`
- 说明：本指令用于「波形模式（持续上传模式）」下获取测量数据。
- 回复为 **§4.6 的测量数据上传帧（15 字节）**。原文档给出的三类状态示例：

| 状态 | 回复帧（原文档示例） | 解读 |
|---|---|---|
| 无手指无数据 | `55 B0 00 00 00 00 00 00 00 00 00 00 B1 AA` | Status=0x00 |
| 有手指无数据 | `55 B0 01 00 00 00 00 00 00 00 00 00 B1 AA` | Status=0x01 |
| 有手指有数据 | `55 B0 02 62 4B 3E 00 00 00 00 00 00 00 AA` | Status=0x02；SpO₂=0x62=98；HR=0x4B=75；PI=0x3E=62→6.2% |

> ⚠️ **示例帧与 §4.6 公布结构存在差异**：
> 1. 上述示例帧为 **14 字节**，而 §9.3 公布的测量数据帧为 **15 字节**（含独立 CheckSum 字节 + Tail `0xAA`）。
> 2. 示例中的 CheckSum 字节值与 §9.3 的累加和公式**对不上**（例 1 末字节 `B1` 按公式应为 `B0`；例 3 末字节 `00` 按公式应为 `0x9D`）。
> 3. ⚠️ **原文 §9.3 与 §9.7 对 OP_Code=0x03 适用模式表述矛盾**：§9.3 步骤 3 称「获取测量数据用于**询问-响应工作模式**下，定时发送模式和波形模式跳过此步骤」；§9.7 称「本指令用于**波形模式**下获取测量数据使用」。两处冲突。**工程建议**：询问-响应模式用 0x03 主动查询；定时/持续上传模式由模块主动上发 15 字节帧，是否仍需 0x03 拉取请以模块实际固件行为为准。
> 工程实现请以 **§4.6 公布的 15 字节结构与校验和公式**为准，并**以模块实际返回逐字节核对**；不要直接照搬 §9.7 示例里的校验和数值。

#### 4.5.4 写 EEPROM（OP_Code = 0x04）

- 格式：`55 B1 04 [Addr] [Data_H] [Data_L] [CS] AA`
- 示例（写 ModeConfig=0x00，定时发送+红光灭，详见 §8）：`55 B1 04 01 00 00 B6 AA`
- 校验和：`B1+04+01+00+00=0xB6` ✓。

#### 4.5.5 读 EEPROM（OP_Code = 0x05）

- 格式：`55 B1 05 [Addr] 00 00 [CS] AA`
- 模块回复对应地址的 EEPROM 数据帧（回显 + 数据）。
- 示例（读 ModeConfig 0x01）：`55 B1 05 01 00 00 B7 AA` → 回复含当前值。
- 校验和：`B1+05+01+00+00=0xB7` ✓。

#### 4.5.6 校准侦测手指灵敏度（OP_Code = 0x06）

- 命令：`55 B1 06 00 00 00 B7 AA` → 回复 `55 B1 06 00 00 00 B7 AA`
- **用途**：校准手指侦测灵敏度。将手指放在需要感应到手指的位置，保持手指相对稳定，发送此命令即可完成校准。
- ⚠️ **侦测距离不应超过 2 cm**，否则会对测量有影响。
- 校验和：`B1+06+00+00+00=0xB7` ✓。

### 4.6 UART 测量数据上传帧（§9.3）

**适用场景**：定时发送模式 / 持续上传（波形）模式下，模组主动上发的测量数据。

**帧结构（15 字节，含 Head 与 Tail）**：

| Byte | 字段 | 含义 | 范围 / 单位 |
|---|---|---|---|
| 0 | `0x55` | Head | 帧头 |
| 1 | `0xB0` | CMD | 数据帧命令字（**注意区别于命令帧的 0xB1**） |
| 2 | Status | 状态 | 见 §7.1 |
| 3 | SpO₂ | 血氧 | 35–99 % |
| 4 | Heart Rate | 心率 | 30–250 BPM |
| 5 | PI | 血流灌注指数 | 0–200，1 LSB = 0.1% |
| 6 | HRV | 心率异变性 | 0–255 ms |
| 7 | 脉搏高度 | 脉搏波幅度 | 0–32 |
| 8 | 心脏收缩时间 | — | 0–255 ms |
| 9 | PPG 波形1 ADC1_H | 红外光波形高字节 | — |
| 10 | PPG 波形1 ADC1_L | 红外光波形低字节 | — |
| 11 | PPG 波形2 ADC2_H | 红光波形高字节 | — |
| 12 | PPG 波形2 ADC2_L | 红光波形低字节 | — |
| 13 | CheckSum | 校验和 | `(Byte1 + … + Byte12) & 0xFF` |
| 14 | `0xAA` | Tail | 帧尾 |

```
CheckSum(Byte13) = (Byte1 + Byte2 + Byte3 + Byte4 + Byte5 + Byte6 + Byte7
                    + Byte8 + Byte9 + Byte10 + Byte11 + Byte12) & 0xFF
```

> ⚠️ 原文档 §9.7 的回复示例帧长度与上式结构不一致（见 §4.5.3 标注），实现时**以本表 15 字节结构为准**。
> **PPG 波形还原**：`ADC1 = ADC1_H×256 + ADC1_L`（红外），`ADC2 = ADC2_H×256 + ADC2_L`（红光），为 16 位 ADC 原始值，可用于自行绘制脉搏波。

### 4.7 UART 工作模式配置与查询

工作模式由 EEPROM `01h`（ModeConfig）决定。UART 提供专用配置/查询命令（对应 OP_Code 0x04 写、0x05 读）。

#### 4.7.1 配置工作模式（§9.10，八种组合）

需配置 ModeConfig，命令格式 `55 B1 04 01 00 [Data_L] [CS] AA`，其中 `Data_L` 的编码见 §8。原文档八种配置如下：

| M_DATA1(b2) | M_DATA0(b1) | M_DETECT(b0) | 模式描述 | 串口指令（含 CS） |
|---|---|---|---|---|
| 0 | 0 | 0 | 定时发送模式，红光灭 | `55 B1 04 01 00 00 B6 AA` |
| 0 | 0 | 1 | 定时发送模式，红光亮 | `55 B1 04 01 00 01 B7 AA` |
| 0 | 1 | 0 | 询问-响应模式，红光灭 | `55 B1 04 01 00 02 B8 AA` |
| 0 | 1 | 1 | 询问-响应模式，红光亮 | `55 B1 04 01 00 03 B9 AA` |
| 1 | 0 | 0 | 持续上传模式，红光灭 | `55 B1 04 01 00 04 BA AA` |
| 1 | 0 | 1 | 持续上传模式，红光亮 | `55 B1 04 01 00 05 BB AA` |
| 1 | 1 | 0 | 定时发送模式，红光灭 | `55 B1 04 01 00 06 BC AA` |
| 1 | 1 | 1 | 定时发送模式，红光亮 | `55 B1 04 01 00 07 BD AA` |

> ⚠️ **配置注意（原文档原文）**：「表格命令默认 CTRL_LOGIC=0，M_PLUSE=0，若不全为 0，应先将 ModeConfig 读取出来，保留数据的 Bit7~Bit3，根据需要配置 Bit2~Bit0，再将更改后的数据保存到 ModeConfig 存储器。」即**读-改-写**策略，不要直接覆盖整字节。

#### 4.7.2 查询工作模式信息（§9.11）

- 命令：`55 B1 05 01 00 00 B7 AA`
- 回复为 ModeConfig 当前值帧，按 `M_DATA1/M_DATA0/M_DETECT` 三位解析（见 §8）。例如回复 `55 B1 05 01 00 07 BE AA` → `Data_L=0x07` → M_DATA1=1, M_DATA0=1, M_DETECT=1 → 定时发送、红光亮。

### 4.8 UART 光强与发送间隔（§9.12–§9.21）

所有「设置/查询」命令结构一致：`55 B1 04 [Addr] [DH] [DL] [CS] AA`（设置）与 `55 B1 05 [Addr] 00 00 [CS] AA`（查询）。

**光强规则（通用）**：`光强 = Data_H×256 + Data_L`；灵敏度**仅与红外光最小/最大光强有关，与红光无关**（原文档反复强调）。

| 功能 | Addr | 设置范围 | 示例（设置值） | 原文示例帧 |
|---|---|---|---|---|
| 设置红外光最小光强 | 02h | 0 ~ IRMaxLevel（越界则钳为 IRMaxLevel） | 17000 | `55 B1 04 02 42 68 61 AA` |
| 查询红外光最小光强 | 02h | — | — | `55 B1 05 02 00 00 B8 AA`（回复 `…42 68 62 AA`） |
| 设置红外光最大光强 | 04h | IRMinLevel ~ 65535（越界则钳为 IRMinLevel） | 20000 | `55 B1 04 04 4E 20 27 AA` |
| 查询红外光最大光强 | 04h | — | — | `55 B1 05 04 00 00 BA AA`（回复 `…4E 20 28 AA`） |
| 设置红光最小光强 | 06h | 0 ~ REDMaxLevel（越界则钳为 REDMaxLevel） | 17000 | `55 B1 04 06 42 68 65 AA` |
| 查询红光最小光强 | 06h | — | — | `55 B1 05 06 00 00 BC AA`（回复 `…42 68 66 AA`） |
| 设置红光最大光强 | 08h | REDMinLevel ~ 65535（越界则钳为 REDMinLevel） | 20000 | `55 B1 04 08 4E 20 2B AA` |
| 查询红光最大光强 | 08h | — | — | `55 B1 05 08 00 00 BE AA`（回复 `…4E 20 2C AA`） |
| 设置定时发送间隔 | 0Ah | 1 LSB=4ms；Min=12ms（越界则钳为 12ms） | 1000ms（0x00FA） | `55 B1 04 0A 00 FA B9 AA` |
| 查询定时发送间隔 | 0Ah | — | — | `55 B1 05 0A 00 00 C0 AA`（回复 `…00 FA BA AA`） |

> 数值验算：17000 = `0x42×256 + 0x68` = 66×256+104 = 17000 ✓；20000 = `0x4E×256+0x20` = 78×256+32 = 20000 ✓；1000ms = `0xFA × 4ms` = 250×4 = 1000ms ✓。
> ⚠️ **定时发送间隔仅对 UART 定时发送模式生效**（EEPROM 说明中标注「用于 UART 接口场景」），IIC 接口不走此定时机制。

### 4.9 UART 标准操作步骤（对应原文 §9.3 框架）

原文档 §9.3 给出一套「模块操作步骤」六步框架，是标准使用顺序的权威出处，完整呈现如下：

| 步骤 | 原文操作 | 说明 |
|---|---|---|
| 1. 上电 | 出厂默认模式为：**按客户需求**；上电工作状态：**开始测量** | 工作模式由 EEPROM 出厂 ModeConfig 决定，未必是默认定时发送（见 §8、§11.2） |
| 2. 开始测量 | — | 参见 §9.4 / §10.4 |
| 3. 获取测量数据 | 用于**询问-响应工作模式**下；定时发送模式和波形模式跳过此步骤 | 见 §4.5.3 模式矛盾说明 |
| 4. 结束测量 | 参见第 9 节（原文误写「第 8 节」） | 参见 §9.5 |
| 5. 进入休眠 | 参见第 9 节（原文误写「第 8 节」）；休眠功耗 110µA 左右，若对功耗有要求，应**失能模块**（拉低 EN） | 参见 §9.6 |
| 6. 重复 2~5 步骤 | 若需要重新配置模块工作模式，可在**任意步骤**进行配置 | 模式可**热切换**，无需重启 |

> ⚠️ **原文章节引用笔误**：步骤 4/5 原文写「参见第 8 节」，但结束测量 / 进入休眠均属 **§9 UART Operation**，应为「**第 9 节**」。
> **热切换提示（B2）**：步骤 6 明确工作模式可在任意时刻通过写 ModeConfig（01h）重配，无需重启模块。

## 5. IIC 通信协议

### 5.1 IIC 时序符号与从机地址

| 符号 | 含义 |
|---|---|
| **S** | IIC Start（起始条件） |
| **W** | Write，地址/方向位 = 0 |
| **R** | Read，地址/方向位 = 1 |
| **A** | ACK（应答，0 为 ACK，1 为 NACK） |
| **P** | IIC STOP（停止条件） |
| **Sr** | 重复起始（Repeated Start） |
| `→` 主→从 | Master-to-Slave |
| `→` 从→主 | Slave-to-Master |

**7 位从机地址 = `0x64`**：
- 写地址字节 = `0x64 << 1 | 0` = `0xC8`
- 读地址字节 = `0x64 << 1 | 1` = `0xC9`

> ⚠️ 原文档多处把从机地址写作 `0x064`（如 §10.2 / §10.3 / §10.4 / §10.5 / §10.6 的时序图），多写了一个前导 0，**正确 7 位地址为 `0x64`**（10.1 写入示例的 `i2c_write(0x64)` 才是正确的）。`i2c_write(0x65)` 出现在「读」流程里，是库函数把「7 位地址 0x64 + 读方向(1)」传参为 `0x64|1 = 0x65`，函数内部会再左移拼接，属同一地址，并非新地址。

### 5.2 IIC 命令总表

| Command | 功能 | 传输方向 | 参数 | 返回 |
|---|---|---|---|---|
| `0x00` | 读取测量数据 | W → Sr → R | 无 | Status, SpO₂, HR, PI, HRV, **PEC**（6 字节） |
| `0x01` | 设置模块工作状态 | W | OP_Code（0=开始，1=停止，2=休眠） | 无 |
| `0x03` | 写入 EEPROM | W | Addr + Data_H + Data_L | 无（Data_H→Addr，Data_L→Addr+1） |
| `0x04` | 读取 EEPROM | W → Sr → R | Addr | Data_H(Addr), Data_L(Addr+1) |
| `0x05` | 模块灵敏度校准（纯写） | W | 无 | 无 |
| `0x05` | 获取相关信号（测试用，写后读） | W → Sr → R | 无 | IR_DC_H, IR_DC_L, RED_DC_H, RED_DC_L, 0x00, **PEC**（6 字节） |

> ⚠️ **Command `0x05` 被两个功能复用**：「灵敏度校准」是纯写（无返回），「获取相关信号」是写后读（返回 6 字节），靠**传输方向**区分。原文档未给出 IIC 的「更新波形控制参数」「获取 PPG 波形」等命令（见第 6 章差异矩阵）。
> ⚠️ **Command `0x02` 在 IIC 中未定义**（UART 的 0x02 用于更新波形控制参数，IIC 无对应命令）。

### 5.3 IIC 写入 EEPROM（Command 0x03）

```
S | 0x64(W) | A | 0x03 | A | Addr | A | Data_HighByte | A | Data_LowByte | A | P
```

- `Data_HighByte` 写入地址 `Addr`，`Data_LowByte` 写入地址 `Addr+1`。
- 示例（写地址 0x00/0x01 为 0x55/0x56）：

```c
i2c_start();
i2c_write(0x64);  i2c_ack();
i2c_write(0x03);  i2c_ack();
i2c_write(0x00);  i2c_ack();   // Addr
i2c_write(0x55);  i2c_ack();   // Data_HighByte -> 0x00
i2c_write(0x56);  i2c_ack();   // Data_LowByte -> 0x01
i2c_stop();
```

### 5.4 IIC 读取 EEPROM（Command 0x04）

```
S | 0x64(W) | A | 0x04 | A | Addr | A | Sr | 0x64(R) | A | Data_HighByte | A | Data_LowByte | A | P
```

- `Data_HighByte` = 地址 `Addr` 的数据，`Data_LowByte` = 地址 `Addr+1` 的数据。
- 示例（读 0x00/0x01）：

```c
i2c_start();
i2c_write(0x64);  i2c_ack();
i2c_write(0x04);  i2c_ack();
i2c_write(0x00);  i2c_ack();   // Addr
i2c_write(0x65);  i2c_ack();   // Sr + Read(0x64|1=0x65)
DATA_H = i2c_read(ACK);
DATA_L = i2c_read(ACK);
i2c_stop();
DATA = DATA_H * 256 + DATA_L;   // 修正后：见 §10 注释
```

> ⚠️ **原文档代码陷阱**：示例末行写作 `DATA = DATA_H << 8 + DATA_L;`。在 C 语言中 `+` 优先级**高于** `<<`，该式等价于 `DATA_H << (8 + DATA_L)`，是 **Bug**。应改为 `DATA = (DATA_H << 8) + DATA_L;` 或 `DATA_H * 256 + DATA_L`，与原文档EEPROM说明的「高字节×256+低字节」一致。

### 5.5 IIC 读取测量数据（Command 0x00）

```
S | 0x64(W) | A | 0x00 | A | Sr | 0x64(R) | A | Status | A | SpO2 | A | HR | A | PI | A | HRV | A | PEC | A | P
```

- 返回 **6 字节**：`Status, SpO2, Heart_Rate, PI, HRV, PEC`。
- 状态枚举：`0x00`=无手指；`0x01`=有手指无数据；`0x02`=有手指有数据。
- **PEC 公式**：`PEC = (Status + SpO2 + Heart_Rate + PI + HRV) & 0xFF`
- 示例：

```c
i2c_start();
i2c_write(0x64);  i2c_ack();
i2c_write(0x00);  i2c_ack();
i2c_write(0x65);  i2c_ack();   // Sr + Read
Status     = i2c_read(ACK);
SpO2       = i2c_read(ACK);
Heart_Rate = i2c_read(ACK);
PI         = i2c_read(ACK);
HRV        = i2c_read(ACK);
PEC        = i2c_read(ACK);
i2c_stop();
if (PEC == ((Status + SpO2 + Heart_Rate + PI + HRV) & 0xFF)) {
    Data = OK;
}
```

> ⚠️ 原文档 PEC 公式变量名写作 `Staus`（拼写应为 `Status`），且公式中写 `Heart Rate`（空格）而代码用 `Heart_Rate`——语义相同，实现时以 5 字节之和为准。

### 5.6 IIC 设置模块工作状态（Command 0x01）

```
S | 0x64(W) | A | 0x01 | A | OP_Code | A | P
```

- `OP_Code`：`0x00`=开始测量；`0x01`=停止测量；`0x02`=进入休眠。
- 示例：

```c
// 开始测量
i2c_start(); i2c_write(0x64); i2c_ack();
i2c_write(0x01); i2c_ack(); i2c_write(0x00); i2c_ack(); i2c_stop();
// 停止测量
i2c_start(); i2c_write(0x64); i2c_ack();
i2c_write(0x01); i2c_ack(); i2c_write(0x01); i2c_ack(); i2c_stop();
// 进入休眠
i2c_start(); i2c_write(0x64); i2c_ack();
i2c_write(0x01); i2c_ack(); i2c_write(0x02); i2c_ack(); i2c_stop();
```

> 注：原文档 §10.4 标题下方附带了一张 `Status/SpO2/HR/PI/HRV/PEC` 的位图，实质是 **§10.3 读测量数据的返回字节布局**（被排版放在 §10.4 区域）。IIC「设置工作状态」本身是**纯写、无数据返回**，不要误以为会返回测量数据。

### 5.7 IIC 模块灵敏度校准（Command 0x05·写）

```
S | 0x64(W) | A | 0x05 | A | P
```

```c
i2c_start();
i2c_write(0x64);  i2c_ack();
i2c_write(0x05);  i2c_ack();
i2c_stop();
```

- 用途同 UART §4.5.6：手指放好、稳定后下发完成校准。

### 5.8 IIC 获取相关信号 / 测试用（Command 0x05·读）

```
S | 0x64(W) | A | 0x05 | A | Sr | 0x64(R) | A | IR_DC_H | A | IR_DC_L | A | RED_DC_H | A | RED_DC_L | A | 0x00 | A | PEC | A | P
```

- 返回 **6 字节**：`IR_DC_H, IR_DC_L, RED_DC_H, RED_DC_L, 0x00(保留), PEC`。
- 示例：

```c
i2c_start();
i2c_write(0x64);  i2c_ack();
i2c_write(0x05);  i2c_ack();
i2c_write(0x65);  i2c_ack();   // Sr + Read
IR_DC_H  = i2c_read(ACK);
IR_DC_L  = i2c_read(ACK);
RED_DC_H = i2c_read(ACK);
RED_DC_L = i2c_read(ACK);
Temp     = i2c_read(ACK);      // 固定 0x00
PEC      = i2c_read(ACK);
i2c_stop();
```

> ⚠️ **原文档 PEC 公式笔误（重要）**：原文写 `PEC = (IR_DC_H + IR_DC_L + RED_DC_H + RED_DC_H + 0x00) & 0xFF`，其中 `RED_DC_H` **重复出现、漏掉了 `RED_DC_L`**。
> **修正后应为**：`PEC = (IR_DC_H + IR_DC_L + RED_DC_H + RED_DC_L + 0x00) & 0xFF`。实现时务必使用修正公式，否则 PEC 校验会失败。

## 6. 双接口功能覆盖差异矩阵

> **专家洞察**：两套总线并不等价。下表汇总「哪些功能 UART 有专用命令、哪些 IIC 能间接做、哪些 IIC 完全缺失」。

| 功能 | UART 命令 | IIC 命令 | 说明 |
|---|---|---|---|
| 开始测量 | OP 0x01 / Data=00 | Cmd 0x01 / OP=00 | 等价 |
| 结束测量 | OP 0x01 / Data=01 | Cmd 0x01 / OP=01 | 等价 |
| 进入休眠 | OP 0x01 / Data=02 | Cmd 0x01 / OP=02 | 等价 |
| 更新波形控制参数 | OP 0x02 | ❌ 无命令 | IIC 不传波形，无需此命令 |
| 获取生理测量结果 | OP 0x03（15 字节全量帧） | Cmd 0x00（6 字节：Status/SpO₂/HR/PI/HRV） | IIC 不返回脉搏高度/收缩时间/PPG波形 |
| **获取 PPG 波形（ADC1/ADC2）** | ✅ 上传帧含 | ❌ 仅 Cmd 0x05 返回**直流分量** IR_DC/RED_DC | **实时波形只能走 UART** |
| 读 EEPROM | OP 0x05 | Cmd 0x04 | 等价 |
| 写 EEPROM | OP 0x04 | Cmd 0x03 | 等价 |
| 校准手指灵敏度 | OP 0x06 | Cmd 0x05（写） | 等价 |
| 配置工作模式 | OP 0x04 写 01h（8 种直配） | Cmd 0x03 写 01h（间接） | IIC 需自行组 ModeConfig 字节 |
| 查询工作模式 | OP 0x05 读 01h | Cmd 0x04 读 01h | 等价 |
| 设置/查询 IR 光强 | OP 0x04/0x05 读写 02h/03h | Cmd 0x03/0x04 读写 02h/03h | IIC 间接可达 |
| 设置/查询 RED 光强 | OP 0x04/0x05 读写 06h–09h | Cmd 0x03/0x04 读写 06h–09h | IIC 间接可达 |
| 设置/查询发送间隔 | OP 0x04/0x05 读写 0Ah/0Bh | Cmd 0x03/0x04 读写 0Ah/0Bh | IIC 可写但**间隔仅 UART 定时模式生效** |
| 获取原始光强直流分量（测试） | ❌ 无 | Cmd 0x05（写后读） | UART 无对应命令 |

**关键结论**：
1. **需要实时 PPG 波形** → 必须用 **UART 持续上传模式**。
2. **仅要 SpO₂/HR/PI 数值、且总线受限为 IIC** → 用 IIC Cmd 0x00 读测量数据即可，数值经 PEC 校验更可靠。
3. IIC 用户改工作模式/光强/间隔时，没有专用「配置命令」，需直接 **读写 EEPROM 对应地址**（Cmd 0x03/0x04），并遵循 §8 的 ModeConfig 位定义与「读-改-写」原则。

## 8. 工作模式寄存器 ModeConfig 详解（EEPROM 01h）

### 8.1 位定义

| Bit | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
|---|---|---|---|---|---|---|---|---|
| Name | — | — | — | CTRL_LOGIC | M_PLUSE | M_DATA1 | M_DATA0 | M_DETECT |
| R/W | — | — | — | R/W | R/W | R/W | R/W | R/W |
| Default | 0 | 0 | 0 | 0 | 0 | 0 | 0 | 0 |

### 8.2 各比特含义

- **Bit4 CTRL_LOGIC**（发射管控制逻辑）：
  - `0`：SW_LED1_ON = IR LED 亮，SW_LED2_ON = RED LED 亮（**默认**）
  - `1`：SW_LED1_ON = RED LED 亮，SW_LED2_ON = IR LED 亮
- **Bit3 M_PLUSE**（脉搏波满量程更新模式）：
  - `0`：自发更新（默认，适用于只显示脉搏柱的 LED 屏血氧仪）
  - `1`：响应更新（适用于 OLED/TFT 屏，需配 UART `0x02`「更新波形控制参数」命令避免波形断痕）
- **Bit2~Bit1 M_DATA1~M_DATA0**（数据传输模式）：
  - `00`：**定时发送模式**（默认）
  - `01`：**询问-响应模式**
  - `10`：**持续上传模式**（PPG 波形模式，实时描绘波形）
  - `11`：**定时发送模式**
- **Bit0 M_DETECT**（手指侦测红灯亮灭）：
  - `0`：红灯灭（默认）
  - `1`：红灯亮

> ⚠️ **原文档 §8.1 笔误纠正**：原文在 M_DATA1~M_DATA0 说明中把 `01` 写了两次（「01：定时发送模式(Default)」与「01：询问-响应模式」），且遗漏 `00` 定义。结合 §9.10 八种配置命令（权威），**正确的映射为 `00`=定时发送(默认)、`01`=询问-响应、`10`=持续上传、`11`=定时发送**。本手册已采用修正映射。
> Data_L 字节编码：`Data_L = M_DETECT + (M_DATA0<<1) + (M_DATA1<<2)`（Bit7~Bit5、Bit4=CTRL_LOGIC、Bit3=M_PLUSE 不在模式选择字节内，配置时按 §4.7.1 注「读-改-写」保留）。

## 11. 集成建议与典型流程

### 11.1 选型建议
- **要波形 / 连续监测**：选 **UART + 持续上传模式（M_DATA=10）**。
- **低引脚 / 已被 IIC 总线占用**：选 **IIC + 定时/询问模式**，用 Cmd 0x00 轮询 SpO₂/HR/PI，数据带 PEC 校验更稳。
- **功耗敏感**：不用时拉低 **EN** 引脚（Shut Down 1µA），而非仅用休眠命令（~110µA）。

### 11.2 UART 典型流程（定时发送模式）
```
1. 上电即开始测量；具体工作模式取决于 EEPROM 出厂 ModeConfig（按客户需求定制，未必是默认定时发送，集成前应读取 01h 确认）
2. [可选] 配置工作模式：55 B1 04 01 00 0X B6+X AA   // 见 §4.7.1
3. 持续接收 15 字节上传帧（Head=0x55, CMD=0xB0），按 §4.6 解析并校验 CS
4. 结束测量：55 B1 01 00 00 01 B3 AA
5. 进入休眠 / 拉低 EN
```

### 11.3 UART 典型流程（询问-响应模式）
```
1. 配置为询问-响应：55 B1 04 01 00 02 B8 AA   // M_DATA=01, 红光灭
2. 需要数据时：55 B1 03 00 00 00 B4 AA
3. 解析返回的 15 字节测量帧
```

### 11.4 IIC 典型流程（轮询测量值）
```c
// 启动测量
i2c_write_frame(0x01, 0x00);
// 周期性读测量数据（带 PEC 校验）
i2c_start(); i2c_write(0x64); ack(); i2c_write(0x00); ack();
i2c_write(0x65); ack();          // Sr + Read
Status=i2c_read(ACK); SpO2=i2c_read(ACK); HR=i2c_read(ACK);
PI=i2c_read(ACK); HRV=i2c_read(ACK); PEC=i2c_read(ACK);
i2c_stop();
if (PEC == ((Status+SpO2+HR+PI+HRV)&0xFF)) { /* 数据有效 */ }
```

### 11.5 通用注意事项
- **改 ModeConfig 用读-改-写**：先读 01h 保留 Bit7~Bit3（CTRL_LOGIC/M_PLUSE），再改 Bit2~Bit0。
- **光强越界自动钳位**：设置值超范围会被钳到 Min/Max，不会报错。
- **灵敏度只与红外光强相关**：调红光光强不影响手指侦测灵敏度。
- **先核对固件版本**：V1.6 才支持 IIC，旧版固件命令可能不同（见 §1.2）。

## 12. 常见集成故障排查

> 以下「症状 → 可能原因 → 排查动作」对照，覆盖集成中最常见的失败场景。

| 症状 | 可能原因 | 排查动作 |
|---|---|---|
| 发 UART 命令无任何回复 | ① 固件版本非 V1.6（协议已多次变更）；② 波特率/帧格式错（须 38400/8N1）；③ 校验和算错 | 核对模块丝印版本（§1.2）；确认 38400-8N1；用 §9.1 公式重算 CS |
| 回复帧校验和（CS）不符 | 用了 §9.7 示例里的 CS 值（与 §9.3 公式冲突） | 以 §4.6 的 `CS=(B1..B12)&0xFF` 重算，并以模块实际返回为准 |
| IIC 写/读无 ACK | ① 从机地址错（应为 7 位 `0x64`，写 0xC8/读 0xC9，非原文误写的 0x064）；② SDA/SCL 未上拉（IIC 需 10kΩ 上拉，UART 不需）；③ UART/IIC 引脚冲突同时启用 | 确认地址与方向字节；补上拉；同一时刻只用一种总线 |
| IIC 读测量数据 PEC 校验失败 | 用了未修正的 §10.6 公式（`RED_DC_H` 重复、漏 `RED_DC_L`） | 改用修正公式 `PEC=(IR_DC_H+IR_DC_L+RED_DC_H+RED_DC_L+0x00)&0xFF`（§9.2） |
| 收不到 PPG 波形（ADC 值） | 选了 IIC 接口（IIC 不返回实时波形，仅 UART 上传帧含 ADC1/ADC2） | 改用 UART 持续上传模式（M_DATA=10） |
| 配置工作模式后无变化 | 直接覆盖整字节，清掉了 CTRL_LOGIC/M_PLUSE（Bit7~Bit3） | 用「读-改-写」保留 Bit7~Bit3，只改 Bit2~Bit0（§4.7.1） |
| 校准后手指侦测仍异常 | 校准时手指距传感器 > 2 cm | 重新校准，手指距传感器 ≤ 2 cm（§9.9） |
| 进入休眠后电流仍高（非 ~1µA） | 仅发「进入休眠」命令（~110µA），未拉低 EN | 需拉低 EN 引脚进入 Shut Down（≈1µA，§2.3） |
