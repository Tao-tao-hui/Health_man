# 数据治理体系设计文档

## 1. 概述

### 1.1 文档目的

本文档定义了健康管理项目的数据治理体系设计，旨在建立一套全面、有效的数据治理框架，确保医疗数据的质量、安全、合规性，并将医疗专业知识系统地关联到项目的各项指标中。

### 1.2 适用范围

本设计适用于健康管理项目的所有数据治理活动，包括：
- 数据质量管理
- 数据安全与隐私保护
- 数据生命周期管理
- 数据标准与规范制定
- 数据责任与组织架构
- 医疗专业知识图谱管理
- 指标体系设计与评估

### 1.3 设计原则

| 原则 | 描述 |
|------|------|
| **合规性** | 符合HIPAA、GDPR、《个人信息保护法》等法律法规要求 |
| **完整性** | 覆盖数据治理的全生命周期和核心领域 |
| **专业性** | 指标设计符合医疗行业特性和临床实践标准 |
| **可扩展性** | 支持未来业务发展和技术演进 |
| **可操作性** | 提供明确的工作流程和实施指南 |

## 2. 数据治理框架

### 2.1 治理架构

数据治理框架采用四层架构：

```
┌─────────────────────────────────────────────────────────────┐
│                    战略决策层 (Enterprise)                   │
│  Data Owner / Data Auditor / 数据治理委员会                  │
├─────────────────────────────────────────────────────────────┤
│                    领域管理层 (Domain)                      │
│  Data Steward / Clinical Expert / 业务域专家                 │
├─────────────────────────────────────────────────────────────┤
│                    系统执行层 (System)                      │
│  Data Admin / Data Architect / 技术实现团队                 │
├─────────────────────────────────────────────────────────────┤
│                    数据使用层 (Dataset)                     │
│  Data User / 数据消费者                                    │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 角色定义

| 角色 | 职责 | 权限 | 技能要求 |
|------|------|------|----------|
| **Data Owner** | 数据战略规划、审批重大决策、资源分配 | 审批政策、决策数据策略、授权访问 | 业务战略、风险管理、领导力 |
| **Data Steward** | 质量监控、标准维护、问题协调、培训 | 制定标准、审批质量规则、处理问题 | 数据管理、领域知识、沟通协调 |
| **Clinical Expert** | 验证临床合理性、审核算法准确性、制定知识关联规则 | 审批临床标准、验证指标设计、确认知识图谱 | 临床医学、医学统计学、健康评估 |
| **Data Admin** | 系统基础设施管理、安全策略执行、访问控制维护 | 配置安全设置、管理权限、执行备份恢复 | 系统管理、网络安全、数据库管理 |
| **Data Auditor** | 政策执行审计、质量保证验证、合规检查 | 访问治理记录、执行合规审计、报告发现 | 审计、合规、数据分析 |
| **Data Architect** | 数据架构设计、集成方案规划、接口标准定义 | 设计数据模型、制定接口规范、审批技术方案 | 数据建模、系统架构、ETL设计 |
| **Data User** | 遵守使用规范、报告质量问题、保护数据安全 | 访问授权数据、提交反馈、使用分析工具 | 数据使用、数据解读、合规意识 |

## 3. 医疗专业知识图谱

### 3.1 知识图谱架构

知识图谱由节点、关系、临床标准和版本记录组成：

```
知识图谱
├── 节点 (Nodes)
│   ├── 疾病节点 (Disease)
│   ├── 症状节点 (Symptom)
│   ├── 指标节点 (Indicator)
│   ├── 标准节点 (Standard)
│   ├── 指南节点 (Guideline)
│   ├── 治疗节点 (Treatment)
│   ├── 人群节点 (Population)
│   └── 证据节点 (Evidence)
├── 关系 (Relations)
│   ├── HAS_SYMPTOM - 疾病有症状
│   ├── HAS_INDICATOR - 疾病有指标
│   ├── HAS_STANDARD - 指标有标准
│   ├── BASED_ON - 基于证据
│   ├── REFERENCES - 参考标准
│   ├── APPLIES_TO - 适用于人群
│   ├── INDICATES - 指示疾病
│   ├── RELATED_TO - 相关关系
│   └── RECOMMENDS - 推荐治疗
├── 临床标准 (Clinical Standards)
│   └── WHO标准、AHA标准、BestHealth标准等
└── 版本记录 (Versions)
    └── 知识变更历史记录
```

### 3.2 节点类型定义

| 节点类型 | 描述 | 属性示例 |
|----------|------|----------|
| DISEASE | 疾病定义 | ICD编码、严重程度等级 |
| SYMPTOM | 症状描述 | 症状类型、出现频率 |
| INDICATOR | 健康指标 | 单位、测量方法、数据源 |
| STANDARD | 临床标准 | 参考范围、证据等级 |
| GUIDELINE | 诊疗指南 | 适用人群、推荐方案 |
| TREATMENT | 治疗方案 | 治疗类型、疗效评估 |
| POPULATION | 目标人群 | 年龄范围、性别分布 |
| EVIDENCE | 证据等级 | 研究类型、样本量 |

### 3.3 临床标准库

| 分类 | 范围 (kg/m²) | 描述 |
|------|-------------|------|
| 偏瘦 | < 18.5 | 体重不足 |
| 正常 | 18.5 - 25.0 | 健康体重 |
| 超重 | 25.0 - 30.0 | 需要关注 |
| 肥胖 | >= 30.0 | 需要干预 |

| 分类 | 范围 (%) | 描述 |
|------|----------|------|

| 轻度低氧 | 91 - 94 | 需要关注 |
| 中度低氧 | 86 - 90 | 需要干预 |
| 重度低氧 | <= 85 | 需要紧急处理 |

#### 3.3.3 心率标准 (STD_HEART_RATE)

|------|-----------|------|
| 心动过缓 | <= 59 | 心率过慢 |
| 正常 | 60 - 100 | 正常心率 |
| 心动过速 | >= 101 | 心率过快 |

| 人群 | 分类 | 范围 (%) |
|------|------|----------|

| 男性18-39岁 | 正常 | 11.0 - 16.9 |
| 男性18-39岁 | 警告 | 17.0 - 21.9 |
| 男性18-39岁 | 超重 | 22.0 - 26.9 |
| 男性18-39岁 | 肥胖 | >= 27.0 |
| 女性18-39岁 | 偏瘦 | <= 21.0 |
| 女性18-39岁 | 正常 | 21.0 - 27.9 |
| 女性18-39岁 | 警告 | 28.0 - 34.9 |
| 女性18-39岁 | 超重 | 35.0 - 39.9 |
| 女性18-39岁 | 肥胖 | >= 40.0 |

### 3.4 知识更新与版本控制

知识图谱采用版本控制机制，每次知识更新都会创建版本记录，包含：
- 变更类型（创建、更新、删除）
- 变更内容详情
- 作者和审核者信息
- 审核状态（待审核、已批准、已拒绝）

## 4. 指标体系设计

### 4.1 指标分类

| 分类 | 描述 | 包含指标 |
|------|------|----------|
| BODY_COMPOSITION | 体成分指标 | BMI、体脂率、内脏脂肪等级、肌肉量、骨量 |
| CARDIOVASCULAR | 心血管指标 | 血氧饱和度、心率、灌注指数(PI)、心率变异性(HRV) |
| METABOLIC | 代谢指标 | 基础代谢率(BMR)、BMI、体脂率、内脏脂肪等级 |
| MUSCULOSKELETAL | 肌肉骨骼指标 | 肌肉量、骨量 |
| NUTRITIONAL | 营养指标 | 水分率、蛋白质率 |
| GENERAL_HEALTH | 综合健康指标 | 其他通用指标 |

### 4.2 指标级别

| 级别 | 描述 | 示例 |
|------|------|------|
| PRIMARY | 核心指标，直接采集 | SPO2、心率、体脂率 |
| SECONDARY | 辅助指标，派生计算 | PI、HRV、肌肉量 |
| DERIVED | 推导指标，综合计算 | BMI、BMR |

### 4.5 指标评分与解读

指标评分采用百分制，根据参考范围自动计算：

| 等级 | 评分范围 | 解读 |
|------|----------|------|
| normal | 100 | 处于正常范围，继续保持 |
| warning | 40-70 | 需要关注，建议咨询专业人士 |
| danger | 20 | 异常，建议尽快就医评估 |
| unknown | 50 | 暂无参考标准 |

## 5. 元数据管理

### 5.1 元数据分类

| 类型 | 描述 | 示例 |
|------|------|------|
| DATA_ELEMENT | 数据元素 | 用户ID、身高、体重 |
| DATASET | 数据集 | 用户健康数据集合 |
| TABLE | 数据表 | 用户表、测量记录表 |
| COLUMN | 数据列 | 用户表中的姓名列 |
| INDICATOR | 指标 | BMI、SPO2 |
| SERVICE | 服务 | 数据采集服务 |
| INTERFACE | 接口 | API接口定义 |
| PROCESS | 流程 | 数据处理流程 |
| STANDARD | 标准 | 临床标准定义 |

### 5.2 数据分类级别

| 级别 | 描述 | 示例 |
|------|------|------|
| PUBLIC | 公开数据 | 健康知识、参考标准 |
| INTERNAL | 内部数据 | 设备ID、计算指标 |
| CONFIDENTIAL | 机密数据 | 身高、体重、血氧值 |
| RESTRICTED | 受限数据 | 诊断结果、治疗方案 |
| SENSITIVE | 敏感数据 | 用户ID、姓名、出生日期 |

### 5.3 标签体系

标签分为五类：

| 标签类型 | 描述 | 示例标签 |
|----------|------|----------|
| BUSINESS | 业务标签 | 体成分、心血管、代谢 |
| TECHNICAL | 技术标签 | 实时数据、历史数据、计算数据 |
| QUALITY | 质量标签 | 高质量、质量警告 |
| COMPLIANCE | 合规标签 | PHI、PII、HIPAA合规 |
| SEMANTIC | 语义标签 | WHO标准、临床级 |

### 5.4 数据血缘追踪

数据血缘支持双向追踪：
- **上游追踪**: 从目标数据追溯到数据源
- **下游追踪**: 从数据源追踪到数据消费端

### 5.5 语义映射

语义映射用于建立不同数据模型和标准之间的对应关系，支持：
- 等价映射 (equivalent)
- 包含映射 (subsumes)
- 相关映射 (related)

## 6. 质量保证

### 6.1 质量维度

| 维度 | 描述 | 评估指标 |
|------|------|----------|
| ACCURACY | 数据准确性 | 符合业务规则的比例 |
| COMPLETENESS | 数据完整性 | 字段填充率 |
| CONSISTENCY | 数据一致性 | 跨系统数据一致性 |
| TIMELINESS | 数据及时性 | 数据采集到处理的时间间隔 |
| VALIDITY | 数据有效性 | 值在合理范围内的比例 |
| UNIQUENESS | 数据唯一性 | 无重复记录比例 |
| INTEGRITY | 数据完整性 | 参照完整性检查 |

### 6.2 质量规则类型

| 规则类型 | 描述 | 示例 |
|----------|------|------|
| RANGE_CHECK | 范围检查 | SPO2在70-100之间 |
| FORMAT_CHECK | 格式检查 | 日期格式有效性 |
| NULL_CHECK | 空值检查 | 必填字段非空 |
| UNIQUE_CHECK | 唯一性检查 | 设备ID唯一 |
| CONSISTENCY_CHECK | 一致性检查 | 短时间内数据变化 |
| REFERENTIAL_CHECK | 参照检查 | 外键约束检查 |
| BUSINESS_RULE | 业务规则 | BMI计算准确性 |

### 6.3 质量检查规则示例

| 规则ID | 名称 | 维度 | 字段 | 条件 | 阈值 | 严重程度 |
|--------|------|------|------|------|------|----------|
| RULE_SPO2_RANGE | 血氧饱和度范围检查 | VALIDITY | SPO2 | 70 <= value <= 100 | 0.99 | CRITICAL |
| RULE_HEART_RATE_RANGE | 心率范围检查 | VALIDITY | HEART_RATE | 30 <= value <= 250 | 0.98 | CRITICAL |
| RULE_BMI_RANGE | BMI范围检查 | VALIDITY | BMI | 10 <= value <= 80 | 0.99 | WARNING |
| RULE_SPO2_COMPLETENESS | 血氧完整性检查 | COMPLETENESS | SPO2 | value is not null | 0.95 | WARNING |
| RULE_TIMELINESS | 数据及时性检查 | TIMELINESS | MEASURE_TIME | now - value <= 30min | 0.90 | WARNING |

### 6.4 质量评分体系

| 评分范围 | 级别 | 描述 |
|----------|------|------|
| >= 95% | EXCELLENT | 优秀 |
| 85% - 94% | GOOD | 良好 |
| 70% - 84% | WARNING | 警告 |

## 8. 知识与指标关联技术实现

### 8.1 关联机制

知识与指标的关联通过以下机制实现：

```
医疗知识图谱 ──(HAS_INDICATOR)──→ 指标节点 ──(related_knowledge_ids)──→ 指标定义
    │                                        │
    │                                        │
    └─→ 临床标准 ──(REFERENCES)──→ 参考范围 ──→ 指标评分规则
```

### 8.2 关联方式

| 关联方式 | 描述 | 实现方法 |
|----------|------|----------|
| 元数据关联 | 通过元数据标签建立关联 | 在指标定义中设置related_knowledge_ids |
| 标签体系 | 通过统一标签体系建立关联 | 为知识节点和指标分配相同标签 |
| 语义映射 | 通过语义映射建立关联 | 定义指标与知识概念的映射关系 |
| 血缘追踪 | 通过数据血缘建立关联 | 追踪指标数据的知识来源 |

### 8.3 技术实现

```python
# 指标与知识关联示例
class IndicatorDefinition:
    indicator_id: str
    name: str
    category: IndicatorCategory
    related_knowledge_ids: List[str]  # 关联的知识节点ID列表
    
# 知识节点
class KnowledgeNode:
    node_id: str
    node_type: NodeType  # indicator, disease, standard等
    name: str
    attributes: Dict  # 包含测量方法、单位等信息
    
# 临床标准与参考范围关联
class ClinicalStandard:
    standard_id: str
    reference_ranges: Dict  # 按指标分类的参考范围
    
# 关联查询
def get_related_knowledge(indicator_id):
    """获取指标相关的知识"""
    indicator = indicator_system.get_indicator(indicator_id)
    related_nodes = []
    for knowledge_id in indicator.related_knowledge_ids:
        node = knowledge_graph.get_node(knowledge_id)
        if node:
            related_nodes.append(node)
    return related_nodes
```
