## 1. 概述

### 1.1 项目背景

本项目旨在将芯片硬件（S3008T 体脂测量芯片 + BMH08002 血氧测量模组）的检测能力与临床诊断需求、疾病筛查标准、治疗效果评估体系相结合，构建大健康领域的智能健康监测系统。

### 1.2 核心原则

1. **保健级定位**：所有输出均为保健级参考值，不可用于临床诊断
2. **分层解耦**：硬件感知、数据处理、临床知识、应用服务四层独立演进
3. **标准对接**：整合WHO、AHA等国际标准与中国成人健康标准
4. **尺度控制**：严格控制健康意见输出尺度，避免越界到医疗诊断

### 1.3 目标受众

- **产品经理**：了解系统架构与功能边界
- **工程师**：指导系统实现与集成
- **临床顾问**：验证临床知识的准确性与适用性

## 3. 硬件感知层设计

### 3.1 芯片能力矩阵

| 芯片 | 测量指标 | 精度 | 响应速度 | 输出格式 | 通信协议 |
|:---|:---|:---|:---|:---|:---|
| **S3008T** | 50KHz双脚阻抗 | 1Ω | ~100ms | 大端16位 | UART 9600bps |
| **BestHealth TwoLegs** | 体脂率、BMI、内脏脂肪等19项 | ±3~5% vs DEXA | ~200ms | JSON/API | HTTP |
| **BMH08002** | SpO₂ (70~99%) | ±2% (PI>0.5%) | 1000ms | 15字节帧 | UART/IIC |
| **BMH08002** | HR (30~250BPM) | ±2BPM或±2% | 1000ms | 15字节帧 | UART/IIC |
| **BMH08002** | PI (0.5~25%) | ±0.2%(0~2%)/±1%(2.1~20%) | 1000ms | 15字节帧 | UART/IIC |
| **BMH08002** | HRV、脉搏波、收缩时间 | — | 1000ms | 15字节帧 | UART |

### 3.2 S3008T 通信协议

**帧格式**: `[Length] [CMD] [DataN] [CRC-8]`

**命令清单**:

| 命令号 | 名称 | 说明 |
|:---:|:---|:---|
| 0x01 | 设置测量模式 | Data0: 0=待机, 1=测量 |
| 0x02 | 设置测量频率 | Data0: 0=50KHz |
| 0x03 | 设置体重 | Data0..1: 体重×10(大端16位) |
| 0x04 | 设置身高 | Data0: 身高(cm, 90~220) |
| 0x05 | 设置年龄 | Data0: 年龄(6~99) |
| 0x06 | 设置性别 | Data0: 0=女, 1=男 |
| 0x07 | 设置用户类型 | Data0: 0=普通, 1=运动员 |
| 0x08 | 读取测量结果 | 返回50KHz阻抗值(大端16位) |

**CRC-8 算法**:
- 多项式: `x^8 + x^2 + x + 1` (0x07)
- 初值: 0x00

### 3.3 BMH08002 通信协议

**UART配置**:
- 波特率: 38400 bps
- 数据位: 8 bit
- 校验位: None
- 停止位: 1 bit

**命令帧格式**: `0x55 [CMD] [OP_Code] [Addr] [Data_H] [Data_L] [CheckSum] 0xAA`

**数据帧格式(15字节)**:

| Byte | 字段 | 含义 | 范围/单位 |
|:---:|:---|:---|:---|
| 0 | Head | 帧头 | 0x55 |
| 1 | CMD | 命令字 | 0xB0 |
| 2 | Status | 状态 | 00/01/02 |
| 3 | SpO₂ | 血氧 | 35~99 % |
| 4 | HR | 心率 | 30~250 BPM |
| 5 | PI | 灌注指数 | 0~200(×0.1%) |
| 6 | HRV | 心率变异性 | 0~255 ms |
| 7 | 脉搏高度 | 脉搏波幅度 | 0~32 |
| 8 | 收缩时间 | 心脏收缩时间 | 0~255 ms |
| 9-12 | PPG波形 | ADC原始值 | 16位 |
| 13 | CheckSum | 校验和 | (B1..B12)&0xFF |
| 14 | Tail | 帧尾 | 0xAA |

**IIC配置**:
- 从机地址: 0x64 (7位)
- 命令: 0x00~0x05

### 3.4 关键设计原则

1. **双通道冗余**: S3008T仅UART；BMH08002支持UART/IIC双协议
2. **信号质量评估**: 通信层验证数据完整性(CRC/PEC校验)
3. **低功耗策略**: 非测量时段BMH08002拉低EN引脚(1µA)
4. **固件版本检查**: BMH08002需确认V1.6版本

## 4. 数据处理层设计

### 4.1 数据处理流程

```
原始数据采集 → 协议解析 → 数据校准 → 算法计算 → 质量评估 → 标准化输出
```

### 4.2 体成分数据处理

**输入参数(6项)**:

| 参数 | 标识符 | 分辨率 | 说明 |
|:---|:---|:---:|:---|
| 身高 | heightCm | 1 cm | 直接参与回归模型计算 |
| 体重 | weightKg | 0.1 kg | 四室模型基础项 |
| 50KHz阻抗 | zTwoLegs | 1 Ω | S3008T测量值 |
| 年龄 | age | 1 岁 | 人群分层变量 |
| 性别 | sex | — | M/F，影响回归模型 |
| 用户类型 | userType | — | Normal/Athlete |

**输出参数(19项)**:

| 类别 | 数量 | 指标示例 |
|:---|:---:|:---|
| 全身体成分 | 9 | 水分率、脂肪量、蛋白质率、肌肉量、骨量等 |
| 肥胖分析 | 5 | 脂肪率、内脏脂肪等级、BMI、基础代谢等 |
| 评价建议 | 5 | 身体类型、身体得分、身体年龄、理想体重等 |

**质量评估**:
- 阻抗值范围验证(300~1000Ω)
- 身高/体重合理性检查
- 算法返回值异常检测

### 4.3 血氧数据处理

**原始数据解析**:
- SpO₂: 直接读取(35~99%)
- HR: 直接读取(30~250BPM)
- PI: 读值×0.1(0~20.0%)
- HRV: 毫秒级心率变异性

> ⚠️ **数据范围冲突说明**: 
> - SpO₂范围: 性能指标标注70~99%，通信字节允许35~99%，实际使用以性能指标70~99%为准
> - PI范围: 性能指标标注0.5~25%，通信字节允许0~20.0%，实际使用以通信字节0~20.0%为准
> - 建议在数据处理层添加范围校验，超出性能指标范围的数据标记为可疑数据

**信号质量评估**:
- PI值判断(PI<0.5%时精度下降)
- 信号强度检测
- 运动伪影识别
- 手指接触状态判断(Status字节)

### 4.4 时间序列数据处理

**数据聚合**:
- 实时数据: 单次测量值
- 日/周/月趋势: 滑动窗口统计
- 长期趋势: 指数移动平均

**异常检测**:
- 阈值告警(如SpO₂<90%)
- 趋势突变检测
- 周期性模式识别(如夜间血氧下降)

### 4.5 算法集成方案

**推荐架构**: Web API作为首选方案

```
体脂秤(S3008T) → BLE/Wi-Fi → 云服务(BestHealth API) → 应用层
```

**离线兜底**: Android/iOS SDK

### 5.2 健康评分算法

**多维度评分模型**:

```
综合健康评分 = 体成分评分(50%) + 心血管评分(50%)
```

> ⚠️ **设计说明**: 运动能力评分和生活习惯评分需要外部数据输入（如运动APP、问卷等），当前芯片硬件无法直接测量，暂不纳入核心评分模型。待后续接入外部数据源后可扩展。

**体成分评分计算**:

```
体成分评分 = Σ(单项指标评分 × 权重) / Σ(权重)
其中:
- 体脂率: 权重30%
- BMI: 权重20%
- 内脏脂肪等级: 权重25%
- 肌肉量: 权重25%
```

> ⚠️ **权重说明**: 当前权重分配基于专家共识（证据等级C），后续需通过临床验证数据进行优化调整。

**个性化阈值**:
- 根据年龄、性别、用户类型动态调整标准范围
- 运动员模式使用独立评分标准

## 6. 应用服务层设计

### 6.1 大健康意见输出规范

**输出尺度控制原则**:

| 输出类型 | 允许内容 | 禁止内容 |
|:---|:---|:---|
| 健康评估 | "体脂率偏高，建议关注" | "诊断为肥胖症" |
| 风险预警 | "内脏脂肪等级较高，建议咨询医生" | "确诊为代谢综合征" |
| 改善建议 | "建议每周进行150分钟中等强度运动" | "必须立即就医" |
| 趋势分析 | "近3个月体脂率下降2%" | "病情有所好转" |

**语言风格**:
- 使用温和、鼓励性的语气
- 避免使用医疗诊断术语
- 强调"参考值"、"建议"、"关注"等词汇

**强制免责声明**:

```
本评估结果为保健级参考值，不可用于临床诊断。如有健康疑虑，请咨询专业医疗机构。
```

### 7.3 临床知识层 → 应用服务层

```python
class HealthAssessment:
    """健康评估结果"""
    
    def __init__(self):
        self.overall_score: int = None              # 综合健康评分(0~100)
        self.component_scores: dict = None          # 各维度评分
        self.alerts: list = None                    # 风险预警列表
        self.recommendations: list = None           # 改善建议列表

        self.evidence_level: str = None             # 证据等级
        self.disclaimer: str = "本评估结果为保健级参考值，不可用于临床诊断。"
```
