# 芯片指标与大健康建议融合方案

## 1. 引言

本方案旨在系统梳理芯片硬件检测指标与大健康建议之间的映射关系，设计多维度健康建议生成引擎，实现从芯片原始数据到个性化健康指导的完整闭环。融合方案覆盖BIA体成分指标、心血管指标、TCM体质指标三大维度，确保建议的科学性、个性化和可操作性。

---

## 2. 芯片指标全景图

### 2.1 指标分类体系

芯片硬件可检测的健康指标分为三大类：

| 类别 | 芯片来源 | 指标数量 | 核心指标 |
|-----|---------|---------|---------|
| **BIA体成分指标** | S3008T + TwoLegs算法 | 19项 | 体脂率、BMI、内脏脂肪等级、肌肉量、水分率、蛋白质率、骨量、基础代谢 |
| **心血管指标** | BMH08002 | 5项 | SpO₂、心率、PI、HRV、PPG波形 |
| **TCM体质指标** | 60题问卷 | 9项 | 平和质、气虚质、阳虚质、阴虚质、痰湿质、湿热质、血瘀质、气郁质、特禀质 |

## 3. 指标-建议映射矩阵

### 3.1 映射原则

| 原则 | 说明 |
|-----|------|
| **分级建议** | 每项指标按正常/偏移/异常三档给出不同强度的建议 |
| **证据支撑** | 建议内容基于临床标准和专家共识，标注证据等级 |
| **可操作性** | 建议具体明确，避免模糊表述，包含具体行动步骤 |
| **个性化** | 结合年龄、性别、用户类型动态调整建议内容 |
| **尺度控制** | 严格区分保健级建议与医疗诊断，避免越界 |

### 7.2 核心算法流程

```python
class HealthAdviceEngine:
    """健康建议生成引擎"""
    
    def __init__(self):
        self.single_metric_rules = self._load_single_metric_rules()
        self.cross_metric_rules = self._load_cross_metric_rules()
        self.tcm_fusion_rules = self._load_tcm_fusion_rules()
    
    def generate_advice(self, processed_data: ProcessedData) -> List[Dict]:
        """
        生成健康建议
        
        流程：
        1. 单项指标评估 → 初步建议列表
        2. 交叉指标评估 → 补充交叉建议
        3. TCM融合评估 → 补充中医建议
        4. 聚合去重 → 优先级排序、内容合并
        5. 个性化调整 → 根据用户信息调整建议
        """
        advice_list = []
        
        # 步骤1: 单项指标评估
        advice_list.extend(self._evaluate_single_metrics(processed_data))
        
        # 步骤2: 交叉指标评估
        advice_list.extend(self._evaluate_cross_metrics(processed_data))
        
        # 步骤3: TCM融合评估
        if processed_data.tcm_type:
            advice_list.extend(self._evaluate_tcm_fusion(processed_data))
        
        # 步骤4: 聚合去重
        advice_list = self._aggregate_and_deduplicate(advice_list)
        
        # 步骤5: 个性化调整
        advice_list = self._personalize_advice(advice_list, processed_data)
        
        return advice_list
```

2. **建议映射空间大**：每项指标按正常/偏移/异常三档，可生成99条基础建议；加上交叉建议和TCM融合，建议空间可达数百条
3. **个性化是关键**：年龄、性别、用户类型、体质类型的差异化建议能显著提升建议的针对性和可接受度
