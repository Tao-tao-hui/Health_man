### 2.2 各算法原理与实现

#### 2.2.1 健康评分算法 (HealthScorer)

**原理**：基于临床标准的规则评分，采用加权求和方式

**评分公式**：
```
综合健康评分 = 体成分评分(50%) + 心血管评分(50%)

体成分评分 = Σ(单项指标评分 × 权重) / Σ(权重)
  - 体脂率(30%)、BMI(20%)、内脏脂肪(25%)、肌肉量(25%)

心血管评分 = Σ(单项指标评分 × 权重) / Σ(权重)
  - SpO₂(40%)、心率(30%)、PI(30%)
```

**特点**：
- 基于临床标准知识库([standards.py](file:///e:/Health_man/scripts/clinical/standards.py))进行分类
- 评分区间0~100，易于理解和解释
- 权重分配基于专家共识(证据等级C)

#### 2.2.2 风险预警规则引擎 (RulesEngine)

**原理**：基于临床标准的硬阈值判断

**规则集**：

| 规则ID | 规则名称 | 触发条件 | 严重程度 | 证据等级 |
|--------|---------|---------|---------|---------|
| rule_001 | 内脏脂肪高危预警 | VFAL ≥ 15 | 高 | B |
| rule_002 | 血氧偏低预警 | SpO₂ < 95% | 中 | C |
| rule_003 | 心率异常预警 | HR > 100 或 HR < 50 | 中 | C |
| rule_004 | 肥胖风险预警 | BMI ≥ 28 或体脂超标 | 中 | B |

**特点**：
- 规则可配置，易于扩展
- 包含证据等级标注，便于临床验证
- 支持多规则并行评估

#### 2.2.3 健康风险模型 (HealthRiskModel)

**原理**：基于RandomForestClassifier的3分类模型

**输入特征(12维)**：
```
BIA指标(6维): 体脂率、水分率、肌肉量、BMI、内脏脂肪等级、基础代谢
PPG指标(4维): 心率、RMSSD、SDNN、pNN50
TCM指标(2维): 体质转化分、平和质得分
```

**输出分类**：
- 0: low_risk（低风险）
- 1: medium_risk（中风险）
- 2: high_risk（高风险）

**当前性能（基于合成数据）**：

| 指标 | 基线模型 | 超参数调优 | 特征选择 | 模型压缩 | 集成学习 |
|-----|---------|-----------|---------|---------|---------|
| 准确率 | 68.5% | 70.0% | 68.0% | 66.5% | 68.5% |
| F1分数 | 60.2% | 61.7% | 61.9% | 55.2% | 59.5% |
| 推理时间 | 0.0286ms | 0.0254ms | 0.0268ms | 0.0103ms | 0.0326ms |
| 内存占用 | 2.15MB | 2.17MB | 0MB* | 0.23MB | 2.46MB |

## 3. 性能瓶颈深度诊断

### 3.1 特征维度与数据源匹配缺口分析

**核心问题**：健康风险模型的12维特征中，有4个维度缺乏实际数据源支撑

| 特征维度 | 数据源 | 当前状态 | 说明 |
|---------|-------|---------|------|
| 体脂率 | S3008T BIA | ✅ 已就绪 | 芯片直接输出 |
| 水分率 | S3008T BIA | ✅ 已就绪 | 芯片直接输出 |
| 肌肉量 | S3008T BIA | ✅ 已就绪 | 芯片直接输出 |
| BMI | S3008T BIA | ✅ 已就绪 | 计算得出 |
| 内脏脂肪等级 | S3008T BIA | ✅ 已就绪 | 芯片直接输出 |
| 基础代谢 | S3008T BIA | ✅ 已就绪 | 芯片直接输出 |
| 心率 | BMH08002 SpO2 | ✅ 已就绪 | 芯片直接输出 |
| **RMSSD** | PPG原始波形 | ❌ **需开发** | HRV指标，需从PPG波形计算 |
| **SDNN** | PPG原始波形 | ❌ **需开发** | HRV指标，需从PPG波形计算 |
| **pNN50** | PPG原始波形 | ❌ **需开发** | HRV指标，需从PPG波形计算 |
| **体质转化分** | TCM问卷 | ⚠️ **需接入** | 需从tcm_classifier模块获取 |
| **平和质得分** | TCM问卷 | ⚠️ **需接入** | 需从tcm_classifier模块获取 |

**影响评估**：
- 4个缺失特征占总特征维度的33%
- 缺失特征集中在心血管健康评估维度（HRV指标），直接影响模型对心血管风险的识别能力
- 合成数据无法模拟真实HRV特征的复杂分布，导致模型学习到的模式与真实临床场景存在偏差

### 3.2 数据质量问题

#### 3.2.1 合成数据与真实数据差距

当前[data_generator.py](file:///e:/Health_man/scripts/models/data_generator.py)采用简单正态分布生成数据：

```python
# 当前实现：独立正态分布 + 异常计数
DISTRIBUTION_PARAMS = {
    "body_fat_pct": (22.0, 8.0),
    "heart_rate": (72.0, 12.0),
    ...
}
```

**问题**：
1. 特征间独立分布假设不符合真实生理数据的相关性（如体脂率与BMI高度相关）
2. 异常判定规则过于简单（固定阈值），未考虑年龄、性别等因素
3. 标签生成规则（异常指标数量）无法反映真实临床风险评估逻辑

#### 3.2.2 类别不平衡

从优化报告可见：
- train_low: 185 (23.1%)
- train_medium: 513 (64.1%)
- train_high: 102 (12.8%)

**影响**：模型倾向于预测多数类（medium_risk），对少数类（high_risk）识别能力差

### 3.3 算法体系碎片化问题

**现状**：三套算法体系独立运行，缺乏统一调度和融合机制

```
当前架构（碎片化）:
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ HealthScorer │    │ RulesEngine  │    │HealthRiskModel│
└──────┬───────┘    └──────┬───────┘    └──────┬───────┘
       │                   │                   │
       ▼                   ▼                   ▼
  评分结果              预警列表             风险等级
```

**问题**：
1. 同一指标可能触发多个独立评估，结果不一致
2. 缺乏综合决策层进行加权融合和冲突仲裁
3. 无法充分利用各算法的互补优势

## 4. 融合策略设计

### 4.1 统一评估管道 (Unified Assessment Pipeline, UAP)

**设计理念**：将规则引擎、评分算法、ML模型三层串联，形成统一的健康评估体系

```
统一评估管道架构:
┌─────────────────────────────────────────────────────────────┐
│                    输入层 (ProcessedData)                    │
└───────────────────────────┬─────────────────────────────────┘
                            │
              ┌─────────────┼─────────────┐
              ▼             ▼             ▼
    ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
    │ RulesEngine  │ │ HealthScorer │ │HealthRiskModel│
    │  (即时预警)   │ │ (规则评分)   │ │ (ML风险分级) │
    └──────┬───────┘ └──────┬───────┘ └──────┬───────┘
           │                │                │
           ▼                ▼                ▼
    ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
    │ 预警信息列表  │ │ 多维评分结果  │ │ 风险等级概率  │
    └──────┬───────┘ └──────┬───────┘ └──────┬───────┘
           └────────────────┼────────────────┘
                            │
                            ▼
              ┌──────────────────────┐
              │    综合决策层        │
              │  (加权融合/冲突仲裁) │
              └──────────┬───────────┘
                         │
                         ▼
              ┌──────────────────────┐
              │   HealthAssessment  │
              │  (统一评估输出)      │
              └──────────────────────┘
```

### 4.2 综合决策层设计

**融合策略**：基于证据等级的加权融合

```
综合风险等级 = w₁ × 规则引擎预警 + w₂ × 评分算法结果 + w₃ × ML模型概率

权重分配原则：
- w₁: 规则引擎权重 = 最高证据等级的规则严重程度 × 规则数量
- w₂: 评分算法权重 = 综合健康评分 / 100
- w₃: ML模型权重 = 预测概率(最高风险等级)
```

**冲突仲裁机制**：

| 场景 | 仲裁策略 | 示例 |
|-----|---------|------|
| 规则预警 vs ML预测不一致 | 以高证据等级规则为准 | 规则(证据B)预警高危 vs ML预测低风险 → 输出高危 |
| 评分与ML预测不一致 | 取加权平均值 | 评分70(中) vs ML概率60%(中) → 综合中风险 |
| 多规则冲突 | 取最高严重程度 | 内脏脂肪(高) + 血氧偏低(中) → 输出高风险 |

## 9. 结论与建议

### 9.1 核心发现

1. **特征缺口是最大瓶颈**：12维特征中有4维无实际数据源支撑，直接限制模型性能
2. **数据质量亟待提升**：合成数据无法模拟真实生理特征的复杂分布和相关性
3. **算法体系需要融合**：三套独立算法各自为政，缺乏统一决策机制

### 9.2 优先行动建议

1. **立即启动PPG HRV特征提取开发**：这是提升心血管风险评估能力的关键
2. **实施SMOTE过采样**：快速改善类别不平衡问题，提升高危样本识别率
3. **设计统一评估管道架构**：为后续融合奠定基础
4. **规划真实数据接入路径**：从NHANES等公开数据集开始，逐步积累临床数据
